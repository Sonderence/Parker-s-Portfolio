Research and Preparation

Paligo

Gimmal wanted to re-design and migrate all doc site content from the current host platform to Paligo. I was given two days to learn how to use the software before diving in. I took this time to understand how to create a navigation strucuture, separate topics, and style content.

User Paint Points

User research was done before I started at Gimmal, so we would know what problems needed to be addressed.

  • Users have trouble finding content.
  • It takes too many clicks to get to content.
  • The user might not know website URL.
  • Generally there is only one way to navigate.
  • The flowcharts and images were not styled or were dated.

Empathy Map

I decided to create an empathy map based on these pain points, so I could understand the user better. This served as a mental model that would remind me of the overall goal. I identified the main issue is users ended up having to contact support anyways, which wastes time and eliminates the purpose of having documentation.

Competitive Analysis

We looked at the documentation of large tech companies for design inspiration. We liked the layout of Microsoft's documentation, because they have many products and succesfully created a simple navigation structure for the user.

Process

Using the design guidelines and knowledge given from prior research, I went over the current doc site (all 550+ pages) and created an outline for a new navigation structure.

My Mentor looked over my outline and made a few changes to improve my navigation structure further. We whiteboarded the high-level topic organization for the "Records Management" product, becuase it needed the most revision. I used the whiteboard to keep track of my work and make notes.

Future Doc Site

Zendesk Integration

I proposed the doc site should be integrated with Zendesk. I created a quick mockup in Figma to showcase what this would look like and presented it to my mentor. He, and they product team thought it was a great idea and will execute this upon launch. This will increasee accessability for the user, since the support portal and doc site will be on one page on Gimmal's website.

Persona Based Interface

The idea behind "personas" is the user will have the choice to filter their navigation based on what type of user they are. This will allow even easier navigation and create less noise.

The 3 categories are: End User | Records Manager | Administrator.

Example: An end user won't have any use for documentation describing how to put in PowerShell code.
