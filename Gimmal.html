My Gimmal Internship

Overview
At Gimmal I was tasked with leading the re-design of Gimmal's product documentation. This included designing a new navigation structure, migrating content to a new host platform, and styling/editing the copy. I also proposed integration of the doc site into the IT support platform.

In addition to the doc site project I also wrote a technical blog series, won the social media contest, attended marketing meetings, and presented my final output with the executive leadership team.

Research and Preparation

Paligo

Gimmal wanted to re-design and migrate all doc site content from the current host platform to Paligo. I was given two days to learn how to use the Paligo software before diving in. I took this time to understand how to create a navigation strucuture, separate topics, and style content.

User Paint Points

User research was done to identify user pain points before I started at Gimmal, so we would know what problems needed to be addressed.

  • Users have trouble finding content.
  • It takes too many clicks to get to content.
  • The user might not know website URL.
  • Generally there is only one way to navigate.
  • The flowcharts and images were not styled or were dated.

Empathy Map

I decided to create an empathy map based on these pain points, so I could understand the user better. This served as a mental model that would remind me of the overall goal.

Competitive Analysis

We looked at the documentation of large tech companies for design inspiration. We liked the layout of Microsoft's documentation, because they have many products and succesfully created a simple navigation structure for the user.

Process

Using the design guidelines and knowledge given from prior research, I went over the current doc site (all 550+ pages) and created an outline for a new navigation structure.

My Mentor looked over my outline and made a few changes to improve my navigation structure further. We whiteboarded the high-level topic organization for the "Records Management" product, becuase it needed the most revesion. I used the whiteboard to keep track of my work and make notes.

I migrated what ended up becoming 422 pages of content in an iterative process and learned as I built out the site in Paligo. I also edited and styled the copy as needed.

Future Doc Site

Zendesk Integration

I proposed that the doc site should be integrated with Zendesk. I created a quick mockup in Figma to showcase what this would look like. This will increasee accessability for the user, since the support portal and doc site will be on one page on Gimmal's website.

Persona Based Interface

The idea behind "personas" is the user will have the choice to filter their navigation based on what type of user they are. This will allow even easier navigation and create less noise.

The 3 categories are: End User | Records Manager | Administrator.

Example: An end user won't have any use for documentation describing how to put in PowerShell code.
